# Internal Model Features Paper Outline (Starter Scaffold)

This outline is a compact starting point for collaboratively drafting a paper on internal model features, combining steering-based observations with feature info-page documentation while staying explicit about uncertainty.

## 1. Scope and Framing
- Working question: which internal features are practically useful to study, and why.
- One-open-model-first strategy: start with one open model for end-to-end iteration before broad comparisons.
- Define success as a reproducible evidence trail, not a headline result.
- Keep claims narrow, testable, and revision-friendly.

## 2. Terminology and Boundaries
- Internal features: latent/neuronal representations in the model that may correlate with behaviors.
- Inspection artifacts: tools and pages used to inspect evidence (for example, steering interfaces and Neuronpedia/info pages).
- Explicit distinction: features are properties of the model; tools/pages are observation instruments.
- Note that instrument limitations can shape what evidence is visible.

## 3. Evidence Collection Plan
- For each candidate feature, collect steering evidence from controlled prompt sets.
- For the same feature, collect info-page evidence (summaries, examples, metadata) from Neuronpedia/info pages.
- Record prompts, outputs, and settings in a consistent template.
- Track negative/ambiguous cases alongside supportive cases.

## 4. Analysis and Writing Workflow
- Compare steering and info-page evidence for alignment, mismatch, and uncertainty.
- Use lightweight confidence labels tied to explicit criteria.
- Separate observations from interpretations in draft text.
- Revise sections in short cycles with shared review notes.

## 5. Limits, Risks, and Next Steps
- Avoid over-claiming causality from limited probes.
- Document confounders (prompt sensitivity, sampling variance, interface constraints).
- State replication needs before any broader conclusions.
- End with a prioritized backlog for deeper experiments.
